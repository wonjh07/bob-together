## Next.js 14 + TypeScript Architecture Agent

당신은 Next.js 14 + React 18.x + TypeScript 5.x + Vanilla Extract 앱을 개발하는 코드 생성 에이전트입니다.  
주어진 지침과 규칙에 따라 코드를 작성하고, 프로젝트의 폴더 구조와 아래 규칙을 준수하세요.

---

### 코딩 규칙

2. 공통 컴포넌트를 만들어달라고 지시하면 components에 생성한다.
3. 페이지 관련 코드를 만들어달라고 지시하면 app 폴더에 생성한다
4. 이미지가 첨부되면 참고하여 만든다
5. 불필요한 주석이나 사용하지 않는 코드는 작성하지 않는다.
6. 함수형 컴포넌트를 사용하고, React.FC 타입을 사용하지 않는다
7. 컴포넌트와 함수, 변수 이름은 명확하고 직관적으로
8. 불필요한 중복 코드는 피하고, 재사용 가능한 컴포넌트를 만든다.
9. TypeScript 타입을 명확하게 지정하고, any 타입 사용을 피한다.
10. 컴포넌트 파일명은 camelCase 표기법을 사용한다.
11. 'use client' 지시어가 있는 파일에서 하면 안되는 작업을 하지않는다.
12. Next.js 14의 새로운 기능과 권장 사항을 준수한다.

### 스타일링 규칙

1. 스타일은 하드코딩하지 않고 Vanilla Extract를 사용한다 (className={} 형식으로 적용).
2. 컴포넌트와 스타일 파일은 같은 디렉토리에 위치시킨다. (예: `button.tsx`와 `button.css.ts`)
3. CSS는 flexbox 레이아웃을 기본으로 사용한다.
4. 스타일 속성은 camelCase 표기법을 사용한다.
5. 반응형 디자인을 고려하여 스타일을 작성한다.
6. 스타일 클래스 이름은 컴포넌트 이름과 관련되게 짓는다.
7. 스타일은 가능한 한 재사용 가능하게 작성한다.
8. 컬러는 theme.css.ts 파일에 정의된 변수를 vars.color.~~ 형식으로 사용한다.
9. 셀럭터 사용은 selectors : {} 를 사용한다.
